## 3. 📖 Product definition (locked)

> **Noxioai is an AI-employee platform that lets Iranian businesses run their marketing, social media, customer support, and development from one visual online office — built by TECSO.**

Key strategy decisions:
- **Persian-native as the moat** — not a translated product; natural tone, Jalali calendar, Iranian market context
- **Instagram + Telegram** as first-class channels (what Iranian businesses actually use)
- **Sanctions-resilient AI engine** — open-weight models (Qwen/DeepSeek/Llama family), no dependency on US-restricted APIs
- **Toman pricing via Zarinpal** planned for the commercial phase
- Stack: **Nuxt** frontend (live) · **Go API + PostgreSQL** for Phase B (decided)

## 4. 🎨 Design language (locked)

Dark-first, purple→cyan gradient brand · pixel-art characters as product identity (CSS-only, zero image assets) · glassmorphism shell · motion via @vueuse/motion with `prefers-reduced-motion` respected · SVG icon set (no emoji icons) · accessibility: focus rings, 44px+ touch targets, WCAG-conscious contrast.

## 5. 🔢 Metric

**Phase A: 0 / 100 real waitlist emails in 30 days** — the clock started when the domain went live. Everything is built; the number moves when humans see the link.

## 6. 🧠 Lessons captured (so they're never paid for twice)

1. `npm ci` can reject a lockfile that `npm install` just generated (npm 10/11 self-sync bug with hoisted deps) → CI uses self-healing `npm install` + Node 24
2. GitHub Pages deployments flake (~30% "try again later") → auto-rerun built into the workflow habits; recreating the Pages site via API fixes stuck provisioning
3. Web3Forms free tier is browser-origin only — server-side tests will always fail by design
4. Custom domain switch requires removing the sub-path base URL, or all assets 404
5. RTL must be built from commit #1 — retrofitting costs ~5×
